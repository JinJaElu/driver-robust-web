// 2. 评估主机启动与队列规则

上传数据不会写入 GitHub Pages 页面目录；网关默认固定使用评估主机上的 /dev/shm/driverobust_bridge 作为独立桥接存储区，单文件与总容量均限制为 500MB。如需更换位置，只在评估主机启动服务时用 --fixed-storage-root 固定指定。

cd /home/matrix/seoul_robotics_style/evaluation_v2/pages
python3 local_eval_server.py --host 0.0.0.0 --port 8765

# 如需评估主机管理员固定改到另一个安全位置：
python3 local_eval_server.py --host 0.0.0.0 --port 8765 \
  --fixed-storage-root /固定安全位置/driverobust_bridge \
  --max-upload-mb 500 --storage-limit-mb 500
状态判定方式行为
QUEUEING不存在 <fixed-storage-root>/runtime/SIM_READY 且未设置 DRIVEROBUST_SIM_READY=1合法上传包被保存到 <fixed-storage-root>/eval_queue/，接口返回 202 queued 与排队说明。
READY创建 <fixed-storage-root>/runtime/SIM_READY 或设置环境变量后台 worker 取队列任务，调用 <fixed-storage-root>/runtime/evaluate_job.py 或 .sh。
RETURN任务完成且 manifest 或请求头提供 callback评估主机会将结果 JSON POST 到远端 callback_url，同时保存在 eval_results/。

标记仿真环境就绪 / 暂停接单

# 就绪：开始消费队列
mkdir -p /dev/shm/driverobust_bridge/runtime
touch /dev/shm/driverobust_bridge/runtime/SIM_READY

# 暂停：只接收上传并排队
rm -f /dev/shm/driverobust_bridge/runtime/SIM_READY

// 3. 上传数据包规范（必须满足）

  • 压缩格式：.zip、.tar.gz、.tgz 或 .tar；建议单包不超过评估主机配置上限（默认 500MB）。
  • 根目录必须包含：manifest.json，其 spec_version 必须为 driverobust-eval-upload-v1。
  • 路径安全：压缩包内不得包含绝对路径、.. 回退路径或可执行自启动脚本。
  • 传感器数据：相机图片建议放在 inputs/images/ 或 samples/CAM_*/；点云放在 inputs/lidar/ 或 velodyne/；标定放在 calib/。
  • 标签/预测：真值、预测、扰动配置可放在 labels/、annotations/、predictions/ 或 perturbations/。
  • 结果回传：优先使用请求头 X-Callback-Url；也可在 manifest 中设置 callback_url。
{
  "spec_version": "driverobust-eval-upload-v1",
  "task_id": "remote-task-0001",
  "dataset_type": "kitti",
  "sensor_suite": ["camera", "lidar"],
  "frame_count": 50,
  "files": {
    "images": "inputs/images/",
    "lidar": "inputs/lidar/",
    "calib": "calib/",
    "labels": "labels/",
    "predictions": "predictions/"
  },
  "evaluation": {
    "models": ["yolov5s", "avod", "bevfusion"],
    "metrics": ["recall", "precision", "RDI", "MLD", "CMIF", "DCS", "AJER", "RGI"]
  },
  "callback_url": "https://server.example.com/api/results"
}

远端服务器上传示例

curl -X POST 'http://LOCAL_HOST:8765/api/evaluate/upload?job_id=remote-task-0001' \
  -H 'X-File-Name: driverobust_eval_package.zip' \
  -H 'X-Submitter: remote-server-a' \
  -H 'X-Callback-Url: https://server.example.com/api/driverobust/results' \
  --data-binary @driverobust_eval_package.zip

// 4. 评估执行钩子与结果格式

网关只负责接收、校验、排队、调度与回传。真正仿真评估逻辑建议放在 <fixed-storage-root>/runtime/evaluate_job.py 或 .sh 中，也可放在本文件夹的 runtime/ 中；仿真环境未准备好时不要创建 SIM_READY。

# runtime/evaluate_job.py 会收到：
python3 runtime/evaluate_job.py \
  --job-dir <fixed-storage-root>/eval_queue/<job_id> \
  --archive <fixed-storage-root>/eval_queue/<job_id>/package.zip \
  --result <fixed-storage-root>/eval_results/<job_id>.json

结果 JSON 建议至少包含：

{
  "job_id": "remote-task-0001",
  "status": "completed",
  "metrics": {"recall": 0.718, "precision": 0.64, "RDI": 0.285},
  "artifacts": {"report": "eval_results/remote-task-0001_report.pdf"},
  "completed_at": "2026-07-04T12:00:00Z"
}
