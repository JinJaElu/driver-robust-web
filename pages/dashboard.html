Apollo / Dreamview+ 集成

接口演示、RT 消息映射与可复现入口
系统集成与 Apollo RT 接口设计说明

看板展示的是确定性离线评估与传感器回放结果；如需接入真实 Apollo 环境，可使用障碍物适配器将 YOLO / AVOD / BEVFusion 输出归一化为 PerceptionObstacles， 并通过 CyberRT 发布到 /apollo/perception/obstacles。 每条消息保留 scenario_id、perturbation、模型版本和置信度等元数据，便于在 Dreamview+ 与报告侧做同一场景的失效归因。

RT 集成链路
01离线评估读取图片数据，生成 clean / perturbed 检测结果、逐场景指标和内部障碍物消息。
02Adapter 统一类别、坐标系、3D 尺寸、航向角、速度、置信度与时间戳字段。
03CyberRT 发布节点按固定频率写入 /apollo/perception/obstacles，Dreamview+ 与规划模块可直接订阅。
04看板与回放页面根据同一 scenario_id 对齐指标、图像、点云和注入消息。

后续增强训练建议

基于当前评估结果的优先级排序
01
优先增强扰动数据

夜间眩光、运动模糊、浓雾、遮挡条造成最大 Recall 跌幅，应优先补充这四类扰动的训练样本与数据增强。

02
重点关注 VRU 安全

行人 / 骑行者召回在密集场景下极不稳定，应将密集 VRU 场景单独设为安全关键测试集，并做专项类别重采样训练。

03
建立训练闭环

将// 薄弱场景和低 Recall 场景转为 Hard Examples，结合扰动增强、类别重采样与置信度阈值校准，形成持续改进闭环。

🔍 核心结论

当前 YOLO 对 夜间眩光 (night_glare)、运动模糊 (motion_blur)、浓雾场景 (fog_heavy)、遮挡条干扰 (occlusion_bars) 最为敏感， 这四类扰动导致 Recall 下降 29%—43%。VRU（行人 / 骑行者）是当前感知链路最薄弱的目标类别， 在密集穿越场景下即使无额外扰动也存在严重漏检问题。增强训练应优先补充夜间眩光、 运动模糊、雾天低对比和遮挡样本，并将 VRU 密集场景作为安全关键回归测试集。
